{# Reusable donut chart macro for the Admin Dashboard. Pure SVG, no JS chart library. The caller passes: - ``size`` : pixel width/height of the SVG (e.g. 140) - ``slices`` : list of {label, value, color, href?} dicts. Slices with value 0 are skipped automatically. ``href`` is optional — when present, the slice is wrapped in an ```` element and behaves as a navigation link (per-status drilldown). Colours can be any CSS colour the surrounding context resolves (hex, css var, …). - ``total`` : the total to display in the centre. Caller passes it so an empty pool still renders sensibly (a single grey ring + ``0`` in the centre). - ``stroke`` : ring thickness in px (default 16) Implementation: every slice is a circle whose stroke-dasharray represents (slice_length, gap_remainder) and whose stroke-dashoffset positions the slice along the ring. Browsers render this consistently without needing path-arc math. Renders empty state (single full-circle dim ring) when total <= 0 so operators see "this type has no assets yet" rather than a broken / missing donut. Clickable slices use the SVG ```` element.
