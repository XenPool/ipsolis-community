Recurring background tasks

Read-only view of every Beat-scheduled task in this install — what it does, how often it runs, which queue it lands on, and which app_config knobs influence it. The schedule itself is baked into the worker image; rescheduling requires editing worker/tasks/__init__.py and rebuilding. The Configuration knobs column is where most operational tuning lives — for example, approval reminders respects approval.reminder_after_hours + max_reminders without changing the hourly trigger.
