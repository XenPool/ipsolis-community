Per-classification approval routing

Auto-add an approval step on orders whose asset type carries attributes classified as PII, PHI, or PCI. Two routing modes per class: Compliance officer (one step at the global officer email below — standard for centralised compliance teams) or Owner of record (one step per entry in the asset type's Approval Owners list — standard for HIPAA-style workflows where the data steward who actually owns the regulated surface must sign off, not a generic team). Defaults to none on every class. Activation precedence is strictest-first (PCI > PHI > PII); only one policy fires per order. All steps are de-duped against existing approvers so the same person never gets two approval requests.

SIEM — Audit Log Streaming

Stream every audit_log entry to your SIEM. Adapters today: Splunk HEC, Microsoft Sentinel (Azure Monitor Data Collector API), and a generic HMAC-signed webhook (Elastic, Datadog, Sumo, Loki, homegrown receivers). The worker forwards new rows once a minute, tracks the cursor in app_config, and resumes automatically on transient failure. All back-ends dedupe by event id, so at-least-once delivery is safe.

Splunk setup: Splunk Web → Settings → Data inputs → HTTP Event Collector → New Token. Pick a sourcetype like ipsolis:audit (or leave default — the worker sets it), copy the token. The endpoint URL is your collector, e.g. https://splunk.example.com:8088/services/collector/event.

Sentinel setup: Azure portal → your Log Analytics workspace → Settings → Agents → Log Analytics agent instructions. Copy Workspace ID and Primary key. ip·Solis writes to a custom table called {Log-Type}_CL (default IpsolisAudit_CL) — the table is created on first ingest, no schema registration needed. Uses the HTTP Data Collector API (HMAC-signed); Microsoft supports it through Sept 2026.

Webhook setup: point ip·Solis at any HTTPS endpoint that accepts a JSON array of events. Each POST is signed with HMAC-SHA256 over the raw body and the digest is sent in a configurable header (default X-Hub-Signature-256: sha256=<hex>, GitHub-compatible). Receivers verify by recomputing HMAC with the shared secret. Use the Extra headers field to add per-receiver tokens (Datadog's DD-API-KEY, Sumo's collector URL token, a static Authorization: Bearer …, etc.).

Sent as Authorization: Splunk <token>. Stored as a secret.
